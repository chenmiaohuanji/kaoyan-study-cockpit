# 研途驾驶舱 UI/UX 视觉方案

## 1. 产品体验目标

研途驾驶舱是一套长期、高频使用的个人学习控制台，不是打卡游戏，也不是企业数据大屏。每次打开后，用户应在 10 秒内得到五个答案：

1. 今天最应该先做什么
2. 本周执行节奏是否正常
3. 哪个科目最落后
4. 当前与目标分数相差多少
5. 哪项风险需要立即处理

首页只承载能指导当天行动的数据。纯统计、归因与历史对比进入报告页，降低日常查看时的认知负担。

## 2. 信息架构

| 层级 | 页面 | 核心任务 |
| --- | --- | --- |
| 总览层 | 首页总览 | 快速判断今日行动、整体进度与风险 |
| 执行层 | 今日学习 | 排序任务、进入专注、记录完成质量 |
| 计划层 | 学习计划 | 管理阶段、月、周、日四级计划 |
| 能力层 | 科目进度 | 判断内容、课程、练习、正确率与复习状态 |
| 反馈层 | 错题本、测试成绩 | 找到错误模式、成绩变化与低效投入 |
| 调整层 | 学习报告 | 输出进步、问题、增减建议与下周优先级 |
| 独立训练 | 数学建模 | 独立统计训练线并限制对考研主线的挤压 |
| 系统层 | 设置 | 管理目标、主题、动效与本机数据 |

桌面端使用 232px 固定侧栏与 76px 顶栏；平板侧栏收敛为 84px 图标栏；手机端使用五项底部导航，并将其余模块收进“我的”页和完整菜单。

## 4. 排版规范

- 中文：`PingFang SC`、`HarmonyOS Sans SC`、`MiSans`、`Source Han Sans SC`
- 英文与数字：`Inter`、`SF Pro Display`，当前实现使用 Geist 作为可部署替代
- 页面标题：20px / 650
- 欢迎标题：24px / 650
- 模块标题：17px / 650
- 正文：14px / 400
- 辅助信息：10–12px
- 核心指标：27px / 650
- 专注计时：78px / 550
- 所有字距保持为 0，不随视口宽度缩放字体

数字统一使用稳定字宽，百分比、时长与成绩通过字号和字重突出，不靠高饱和背景突出。

## 5. 空间与表面

- 页面边距：桌面 28px，平板 20px，手机 14px
- 页面模块间距：22px
- 栅格卡片间距：14–18px
- 卡片内边距：17–22px
- 表单间距：14–16px
- 卡片圆角：12–14px
- 控件圆角：8–10px
- 主要按钮高度：40px，大按钮 46–50px
- 输入框高度：42px
- 阴影仅用于弹层与极轻卡片层次；深色模式取消普通卡片阴影

卡片只用于指标、工具和重复项目。页面区段保持无外层卡片嵌套，避免多层边框。

## 6. 核心组件

### 导航

- 活动项采用品牌浅色底、品牌文字和侧边 3px 指示线
- 桌面侧栏底部固定目标、倒计时与主题入口
- 平板仅保留图标并使用 `title` 提示
- 移动端底栏固定：首页、今日、计划、进度、我的

### 指标卡

统一高度，包含标签、核心数值、周期变化、状态说明与一条微型进度。单卡不放复杂图表。

### 任务行

包含完成框、标题、科目与章节、计划时长、优先级、状态与操作。高风险通过细色条与状态标签表达。完成后保留在列表内，降低透明度并显示删除线。

### 状态标签

| 状态 | 颜色 |
| --- | --- |
| 正常信息、学习中、已排期 | 蓝 |
| 已完成、已掌握、达标 | 绿 |
| 需巩固、部分完成、待复习 | 黄 |
| 严重落后、已逾期、已遗忘 | 红 |
| 未开始、无数据 | 灰 |

### 图表

- 一页最多一张主图和一至两张辅助图
- 网格线使用分割线色，仅保留横向参考线
- 主数据使用品牌蓝，成功/关注/风险使用固定语义色
- 折线宽度 2–3px，数据点只在重点或悬停时出现
- 图表必须提供标题、数值 Tooltip 和文字总结
- 环形图仅用于时间构成，不用于精确进度比较

### 表单与反馈

- 输入聚焦使用 1px 品牌描边和 3px 低透明焦点环
- 删除、清空必须二次确认
- 新增、完成、延期、导出通过短时 Toast 反馈
- 无数据时显示线性图标、原因说明和明确行动按钮

### 数据联动

- 新建周计划可选择同步为今日任务，并直接进入专注与完成流程
- 学习记录同时更新章节状态、科目汇总和今日有效时长
- 错题复习反馈会改变掌握状态、复习次数和下一次间隔
- 新测试成绩会更新首页预测分、目标差距、成绩趋势与报告指标
- 目标分调整后，成绩页按各科新目标重新计算达成率
- 所有操作保存到当前浏览器，可通过 JSON 备份导入和导出

## 7. 页面视觉方案

### 首页总览

首屏顺序为欢迎与状态、六项指标、今日关键任务、科目进度、七天趋势和风险。移动端只保留今日完成率、当前任务、开始专注、剩余任务和最严重风险。

### 今日学习

桌面使用任务主列与 310px 专注侧栏。筛选与排序位于顶部工具行。专注模式进入独立全屏层，只保留任务、知识点、计时、暂停、完成与问题记录。

### 学习计划

阶段视图为纵向时间轴，周视图为七列日历，日视图为时间表。移动端将周视图转为纵向七日列表，月视图允许水平滚动。

### 科目进度

顶部为科目切换卡；详情使用七项紧凑指标；章节树以折叠表格展示课程、习题、正确率、复习日期与五级掌握度。复习热力图只做辅助。

### 错题本

桌面为左侧固定筛选和右侧错题卡片。今日复习模式覆盖全屏，一次只显示一道题，先作答再展开解析。

### 测试成绩

先展示总分、目标、差距、变化和稳定/薄弱科目，再展示成绩趋势、目标拆解、失分排行、题型正确率与时间分配。

### 学习报告

报告周期用分段控件切换。自动总结使用品牌浅色横向区域；下方依次为核心指标、投入图表和四类调整建议。

### 数学建模

沿用同一视觉语言但独立统计。时间占比超过预算时使用温和黄色提醒，不使用大面积红色。

## 8. 微交互

- 页面切换：210ms 淡入与 3px 位移
- 卡片悬停：上移 2px并增强边框
- 按钮按下：缩放至 97%
- 进度条：420ms 平滑增长
- 章节展开：160ms 箭头旋转
- Toast：220ms 自下淡入
- 任务完成：勾选框切换为成功色，文本淡化但不立即消失

用户开启“减少动画”或系统启用 `prefers-reduced-motion` 时，所有动画和过渡缩短至近乎即时。

## 9. 响应式规则

| 断点 | 策略 |
| --- | --- |
| `>1220px` | 232px 完整侧栏，12 列内容栅格 |
| `981–1220px` | 84px 图标侧栏，指标三列 |
| `761–980px` | 双列仪表盘，任务与专注改为上下组合 |
| `≤760px` | 隐藏侧栏，固定五项底部导航，单列内容 |
| `≤430px` | 首页只保留首项核心指标，报告洞察单列 |

固定格式组件使用稳定尺寸或最小宽度；表格在窄屏水平滚动，不压缩到文字互相遮挡。

## 10. 无障碍

- 文字和背景按 WCAG 可读对比设计
- 所有图标按钮提供 `aria-label` 或 `title`
- 可交互元素支持键盘焦点
- 状态同时用颜色、文字和形状表达
- 图表容器提供数值语义说明
- 表单标签与输入明确关联
- 支持深色模式与减少动画
- 风险提示保持局部、浅色和可执行，避免焦虑式大面积警告
